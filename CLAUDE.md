# Project Memory & Coordination Protocol

## 1. Governance & Concurrency Safety
- **Primary Architect**: Gemini CLI Conductor.
- **Source of Truth**: Always treat the `plan.md` generated by Conductor as the supreme guide.
- **Concurrency Safety (Read-Before-Write)**: BEFORE any write operation (updating `plan.md`, `tracks.md`, or archiving), you MUST call `read_graph` and `ls -R conductor/` to verify the latest project state. **Never** attempt to "fix" or "backfill" documents if memory indicates they are already `ARCHIVED` or `DONE`.
- **Conflict Resolution**: If any local plan conflicts with the Conductor's `plan.md` or global memory, **stop and ask the user** to reconcile.

## 2. MCP Integration & Sync-Mem Protocol
- **L1-L4 4-Level Sync**: Follow the `sync-mem` protocol for all state changes:
  - **L1 (Graph)**: Update `status`, `progress` (start at 0%), and `last_update` in the `Plan` entity. Link `spec_path` if applicable.
  - **L2-L4 (Files)**: Sync `metadata.json` (L2), `tracks.md` (L3), and `plan.md`/`spec.md` (L4) in parallel.
- **Archive Protocol (Thinning & Auto-Purge)**:
  - **Thinning**: When archiving, remove detailed `todo` and logs from the `Plan` entity; keep only `name`, `status`, `archived_path`, and a final `summary`.
  - **Auto-Purge**: Maintain a maximum of **5 `ARCHIVED` Track entities** in the graph. Delete the oldest ones to prevent memory bloat.
- **Entity Relations** (uppercase):
  - `Project:[ProjectName]` -> `Plan:[ProjectName]` via `HAS_PLAN`
  - `Plan:[ProjectName]` -> `Track:[Name]` via `TRACKS_PROGRESS`
  - `Track:[Name]` -> `Plan:[TrackName]` via `HAS_PLAN`

## 3. Language & Documentation Standards
- **Primary Interaction**: Use **Simplified Chinese (简体中文)** for all user-facing responses and reports.
- **Technical Standards**: **English** is mandatory for all rule-defining files (`GEMINI.md`, `SKILL.md`), technical metadata, and MCP tool logic.
- **Path Rule**: All `file_path` entries MUST be absolute paths.

## 4. Development Standards
- **Coding Style**: Follow the `Style:[ProjectName]` entity properties and `conductor/code_styleguides/`.
- **WSL Environment**: All paths must be WSL-absolute.
