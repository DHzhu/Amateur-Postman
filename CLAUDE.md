# Project Memory & Coordination Protocol

## 1. Governance & Hierarchy
- **Primary Architect**: Gemini CLI Conductor.
- **Source of Truth**: Always treat the `plan.md` generated by Conductor as the supreme guide.
- **Conflict Resolution**: If any Claude-generated plan (`ROADMAP.md`, `PHASE_X_PLAN.md`) conflicts with the Conductor's `plan.md`, **stop and ask the user** to reconcile the "Source of Truth".

## 2. MCP Integration (server-memory)
- **Startup Action**: Before every task, call `server-memory:search_nodes` to find entities starting with `Plan:` or `Project:`.
- **Active Plan**: Locate the `file_path` property of the `Plan:[ProjectName]` entity and read that Markdown file before proposing changes.
- **Progress Tracking**: After completing a task or Phase, you MUST update the `status` and `last_update` properties in `server-memory`.

## 3. Collaborative File Management
- **ROADMAP.md**: Use this only as a high-level progress dashboard for Claude Code. It must sync with the milestones in Conductor's `plan.md`.
- **PHASE_X_PLAN.md**: These are temporary implementation scratchpads. Once a phase is merged, archive or delete these to reduce clutter.
- **Project Context**: Reference `LastContext` in memory to understand previous work done in Gemini CLI or Antigravity IDE.

## 4. Development Standards
- **Coding Style**: Follow the `Style:[ProjectName]` entity properties found in memory.
- **WSL Environment**: All paths must be WSL-absolute.

## 5. Communication & Language Preferences
- **Primary Language**: Always respond and interact in **Simplified Chinese (简体中文)**.
- **Technical Context**: You may process English documentation (e.g., Conductor plans, specs) and use English for MCP tool calls, but the user-facing explanation must be in Chinese.
- **Output Standard**: Summarize English technical details in Chinese for better clarity.
